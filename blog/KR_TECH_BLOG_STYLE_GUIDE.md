# 한국 기술블로그 레퍼런스 기반 톤/디자인 가이드

## 목표
- 실무 회고형 문체
- 문제/의사결정/검증 중심 구조
- 과장 없는 신뢰형 카피

## 톤 가이드
1. 문제를 먼저 쓴다 (왜 이 작업을 했는지)
2. 선택지와 트레이드오프를 분리한다
3. 결과는 수치/검증으로 표현한다
4. 과장 대신 재현 가능한 절차를 남긴다

## 기본 포스트 구조
1) 작업 일자/배치
2) 요약
3) 배경/문제
4) 증상과 제약
5) 선택지 비교
6) 최종 결정/실행
7) 결과
8) 체크리스트
9) 다음 액션

## 페이지 구성 가이드
- 홈: 히어로 + 최근 배치 카드 + 카테고리/태그
- 글 목록: 카드형 + 날짜 + 요약 + 태그
- 글 상세: 상단 메타 + 본문 + 하단 이전/다음/목록

## 디자인 원칙
- 가독성 우선 (line-height 1.7~1.8)
- 텍스트 대비 명확 (AA 이상)
- 카드 간격 규칙 통일 (8/16/24 스케일)
- CTA는 한 섹션 1개 핵심만 강조

## 체크리스트
- [ ] 날짜/기준일 노출
- [ ] 계산 근거/출처 표기
- [ ] 재현 명령 포함
- [ ] 면책 문구 포함
- [ ] 다음 포스트 연결 링크
