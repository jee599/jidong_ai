# 집값AI 디자인 트랙 (웹→앱)

## 디자인 방향
- 신뢰형 핀테크 톤 (정확/차분/검증 가능)
- 입력 피로 최소화 + 결과 이해 최대화
- 계산 근거/기준일/면책을 UI에서 항상 보이게

## 웹 IA (정보구조)
1. 홈(검색 + 핵심 계산기 진입)
2. 취득세 계산기 랜딩(SEO)
3. DSR 계산기 랜딩(SEO)
4. 단지 분석 페이지
5. 결제/리포트 페이지

## 레이아웃 원칙
- 상단: 검색 + 현재 기준일 + 신뢰 배지
- 본문: 입력 패널 / 결과 패널 2단(모바일은 1단)
- 하단: 계산 근거 토글 + 출처 + 면책 + 다음 액션

## 앱 전환 계획
- 1단계: 웹 PWA 최적화
- 2단계: Capacitor 래핑(빠른 앱 출시)
- 3단계: 필요 시 React Native 네이티브 전환

## 팔리는 기능 우선순위
P0: 취득세 정확 계산 + 비교표 + PDF
P1: DSR 예상 한도 + 금리 시나리오
P2: 단지 연동 자동입력 + 수익률 3시나리오
P3: 가격 알림(텔레그램) + 저장/공유 카드
